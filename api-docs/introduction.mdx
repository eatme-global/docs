# API Reference

Welcome to our API documentation. Here you'll find comprehensive information about our API endpoints, authentication methods, and best practices for integration.

## Overview

Our API follows REST principles and uses standard HTTP methods. All responses are in JSON format.

## Getting Started

1. [Get your API key](/customer/api-docs/authentication)
2. Make your first API request
3. Explore available endpoints

## Base URL

```bash
https://api.example.com/v1
```

## Authentication

All API requests require authentication using API keys. Include your API key in the Authorization header:

<CodeGroup>

```bash cURL
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.example.com/v1/users
```

```javascript Node.js
const response = await fetch('https://api.example.com/v1/users', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});
```

```python Python
import requests

response = requests.get(
    'https://api.example.com/v1/users',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)
```

</CodeGroup>

## Response Format

All responses follow a consistent format:

```json
{
  "data": {
    // Response data
  },
  "meta": {
    "page": 1,
    "total": 100
  }
}
```

## Rate Limiting

| Plan | Rate Limit | Burst |
|------|------------|-------|
| Standard | 1,000 req/min | 2,000 req/min |
| Enterprise | Custom | Custom |

<Note>
  Exceeding these limits will result in a `429 Too Many Requests` response.
</Note>

```json
{
  "error": {

    "message": "Invalid input provided",
    "details": {
      "field": "email",
      "issue": "must be a valid email address"
    }
  }
}
```
